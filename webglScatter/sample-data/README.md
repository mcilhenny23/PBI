# WebGL Scatter — Sample Data

Three ready-to-drop CSVs generated by `generate-samples.js` (re-run with `node generate-samples.js` if you want fresh randomness).

## 1. `gaussian-50k.csv` — density mode auto-triggers

50,000 points from a standard normal, plus 200 obvious outliers. In Power BI:
- `X` → **X**
- `Y` → **Y**
- `Group` → **Legend**
- `Id` → **Details** (keeps every row identifiable — required for large N)

With **Render mode = Auto** and the default threshold (20,000), the visual switches to a density heatmap. Zoom in to a neighborhood by narrowing the axis via a slicer and it flips back to individual points.

## 2. `two-clusters-30k.csv` — categorical legend + density

Two Gaussian blobs at (-1,-1) and (+1,+1). Great for demoing the legend + density combo. Try **Hexbin** mode with radius 12 for the honest-summary look.

## 3. `small-1k.csv` — pure Points mode

Below the auto threshold — always renders as individual WebGL point sprites. The fastest way to sanity-check that the visual works before dropping in a large dataset.

## The badge

Every render displays `n = X · <mode>` in the corner. This is the honesty feature — the native Power BI scatter silently samples down to ~10k without telling you.

## WebGL not available?

Some corporate GPO configurations disable WebGL. The visual detects this and falls back to Canvas 2D points (capped at 50k), and the badge appends `Canvas fallback (WebGL blocked)`.
